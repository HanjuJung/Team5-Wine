# Wine 🍷
**와인 리뷰 플랫폼 - 와인** 은 와인을 등록하고, 코드잇 부트캠프 와인의 별점을 매겨 사용자들 세계 각국의 다양한 와인을 등록하고 소개하여, 사용자들이 와인을 쉽게 접할 수 있도록 돕는 것을 목표로 합니다.

## 1) 프로젝트 개요


### 🍷 왜 와인 프로젝트를 선정했나요?

와인 프로젝트는 리뷰 작성, 별점 평가, 다양한 모달 같은 여러 컴포넌트를 직접 구현해볼 수 있는 좋은 기회를 제공합니다. 뿐만 아니라, 단순한 로그인 기능을 넘어 구글과 카카오 간편 로그인 기능도 손쉽게 추가해볼 수 있어서, 실제 서비스에 필요한 기술을 배우고 활용할 수 있습니다. 사이트 디자인도 예쁘고 고급스러운 느낌을 주어, 사용자 경험을 한층 더 개선할 수 있다는 점이 이 프로젝트를 선택한 중요한 이유 중 하나였습니다.

### 🍷 어떤 기대 효과가 있을까요?

#### 소비자 경험 향상 및 개인화된 와인 추천
사용자들에게 리뷰 작성과 별점 평가 기능을 통해 개인화된 와인 추천을 제공합니다. <br>
이를 통해 소비자는 자신에게 맞는 와인을 쉽게 찾을 수 있으며, 추천 시스템을 통해 고평가된 와인을 추천받을 수 있습니다.

#### 커뮤니티 기반의 사용자 충성도 증대
사용자들이 직접 리뷰를 작성하고, 와인을 등록하며, 다른 사용자와 소통함으로써 플랫폼에 대한 충성도가 높아지고, 반복 방문이 유도됩니다. <br>
이는 플랫폼의 지속적인 성장과 사용자 수 확대에 기여합니다.

#### 데이터 기반의 마케팅 전략 수립
수집된 리뷰와 별점 데이터는 와인 판매자와 마케터들에게 중요한 자산이 됩니다. <br>
이를 바탕으로 타겟 마케팅, 신제품 개발, 재고 관리 등 데이터 기반의 마케팅 전략을 수립할 수 있습니다.

## 5) 자체 평가 의견


### 🍷 프로젝트를 마치며

#### 윤수현 <br>
와인이라는 특정 주제를 다루며, 다양한 기능을 통해 사용자에게 가치를 제공하는 방법을 공부할 수 있었습니다. 또한, 컴포넌트 사용의 장점을 직접 체감하며, 효율적인 개발의 중요성을 깨달았습니다. 이번 프로젝트에서 얻은 피드백을 바탕으로, 앞으로 더 나은 프로젝트를 개발하기 위해 지속적으로 보완해 나갈 계획입니다.

### 정한주 <br>
강의에서는 '아~ 그렇구나' 하고 넘겼지만, 실제로 적용하는 것에는 차이가 있다는 것을 확실히 알게된 프로젝트였던 것 같습니다.
프로젝트 중 가장 기억에 남는 부분은 next에서 지원하는 <Image>태그 사용에 있어서 어려움을 겪었던 것인데요.
사용에 있어서 어려운 컴포넌트는 아니라고 할 수 있겠지만, 자꾸 내가 원하는대로 렌더링이 안되니 정말 힘들더군요. 
결국 담당 멘토님과 팀원들의 도움을 받아서 해결했지만, 오작동의 이유는 겨우 width, height 프롭이었습니다.(물론 다른 이유도 있었지만)
멘토님께서 멘토링 도중 이야기해주셨던 말인데, 라이브러리와 같은 기능들을 활용할 때, 직관적으로 바로 사용하는 것도 빠르고 좋지만, 그 안에 담겨있는 기능들을 이해하고 쓰는 것과 그냥 가져다 붙여 쓰는 것은 큰 차이가 있다는 것을 느꼈습니다.
팀원들과의 협업하는법, 또 개인적으로 느낀 교훈들까지 얻어가는 게 많았던 프로젝트였습니다.
